# V1.0 Phase Breakdown

## 版本 1：最小可用桌面客户端

目标：

- 一个桌面应用可启动
- 可添加账户
- 可导入 SSH 或 GitHub PAT
- 可绑定仓库
- 可手动生成日报

必须完成：

1. 仓库目录规范
2. Tauri 工程初始化
3. SQLite 模型初始化
4. Stronghold 接入
5. 手动报告生成闭环

禁止提前做：

- 飞书推送
- 周报月报
- 复杂 AI 总结
- 多平台同步

## 版本 2：自动化与可诊断

目标：

- 自动调度
- 运行历史
- 错误诊断
- known_hosts 管理

必须完成：

1. Linux `systemd timer`
2. Windows `Task Scheduler`
3. 错误码体系
4. 历史记录页
5. 首次 SSH 主机确认

## 版本 3：增强能力

目标：

- 多模板
- 高级作者过滤
- AI 总结建议
- 多报告类型

必须完成：

1. 模板管理页
2. 高级时间窗口策略
3. AI 开关和结果隔离
4. 周报 / 管理摘要模板

## 每次进入新版本前必须检查

1. 上一版本的完成标准是否逐条验证
2. 文档是否与当前结构一致
3. 仓库中是否还有临时文件、演示残留、过期原型
4. 是否为本版本定义了明确的“不做什么”
